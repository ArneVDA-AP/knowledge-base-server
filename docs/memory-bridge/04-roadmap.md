## Two grounded bugs found during reassessment
- **`listPendingMemories` hides flagged memories.** `recordMemoryOutcome` sets `review_status='flagged'`
  on a burn, but `listPendingMemories` queries `review_status='pending'` only → burned memories vanish
  from the review queue. **Fix in Pass 3** (widen to `IN ('pending','flagged')`, surfaced in the UI).
- **`v1.js` drops per-agent provenance.** `/api/v1/memory` hardcodes `created_by:'agent'` and never
  records which agent (`req.apiService`) into `author_detail`. **Fix folded into Pass 2.**

## Honesty flags (kept visible)
- Pass 5's conflict heuristic (high cosine + low lexical overlap ⇒ possible contradiction) is Kaiba's own
  composition, **not** independently evidenced; the best invalidation detectors cap at ~55%. Acceptable
  only because it routes to human review, never auto-resolves.
- Pass 6's cross-agent interchange format has no studied prior art; the provenance fix is grounded, the
  format is engineering judgment — it must not lead the roadmap.
- The salience formula's multiplicative confidence/outcome layering is Kaiba's composition; Generative
  Agents validates only the additive recency+importance+relevance core.

## Sequencing rationale
Pass 1 first because 3 and 5 are structurally impossible without per-memory embeddings, and Pass 2's
dedupe-against-existing step is paraphrase-robust only with semantic recall. Pass 2 (auto-consolidation)
fills the pending queue, which makes Pass 3 (the human-dispose UI) urgent — so they ship in that order.
